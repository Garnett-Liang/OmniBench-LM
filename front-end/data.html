测试流程与数据集特点

1. 知识提取

从 Wikipedia 和 Wikidata 获取实体及其属性，提取知识三元组（subject, predicate, object）。

通过 wiki_pl_build.py 等脚本构建 Prolog 规则数据库。

2. 逻辑推理

使用 Prolog 进行逻辑推理，生成衍生事实：

  • 逆向推理：如 "A 是 B 的母亲" → "B 是 A 的子女"
  • 否定推理：如 "A 是 B 的母亲" → "A 不是 B 的父亲"
  • 复合推理：基于多个事实组合形成新事实

3. 基准测试构造

依据推理生成的事实，自动构造 QA 任务：

  • 生成问题（如："谁是 C 的祖父？"）
  • 生成正确答案
  • 生成支持证据

数据集特点

多种知识类别

涵盖文化、地理、历史、人物、健康、数学、自然、社会、技术等领域。

多种推理策略

包括否定推理、逆向推理、复合推理，确保数据集多样性。

自动化数据生成

结合 Prolog 规则和 Wikipedia 语料，自动构造高质量数据集。

可复现性强

所有数据都基于可解释的逻辑推理，确保可验证。
